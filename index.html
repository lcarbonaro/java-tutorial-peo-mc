Variable Type Conversion

(Also known as Casting)

automatic or implicit - when going bigger type

manual or explicit - when going to smaller type

Type casting is when you assign a value of one primitive data type to another type.

In Java, there are two types of casting:

Widening Casting (automatically) - converting a smaller type to a larger type size

byte -> short -> char -> int -> long -> float -> double

Narrowing Casting (manually) - converting a larger type to a smaller size type

double -> float -> long -> int -> char -> short -> byte

one use-case of this involves input most commonly input is of type String but needs to be converted to say int or float will give example

Arrays & ArrayLists

slide notes In Java, an array is a fixed-size collection of elements of the same type. It is a basic data structure with a contiguous block of memory allocated to store its elements. On the other hand, an ArrayList is a dynamic, resizable implementation of the List interface. It is part of the Java Collections Framework and provides more flexibility compared to arrays. Here are some key differences between arrays and ArrayList: Size: Arrays have a fixed size that is determined at the time of declaration. Once an array is created, its size cannot be changed. On the other hand, an ArrayList can grow and shrink dynamically as elements are added or removed. Type: Arrays can store elements of any type (primitive or object). However, ArrayList can only store object references, meaning it can store objects of any type but not primitive types directly. For primitive types, you have to use their corresponding object wrapper classes (e.g., Integer, Character) as ArrayList elements. Arrays are generally suitable for situations where the size is known in advance and doesn't change frequently. ArrayLists are more convenient for scenarios where the size is variable and requires frequent modifications.

Global State
