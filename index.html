            // 1. import the required Java libraries
            import java.io.InputStreamReader;
            import java.io.BufferedReader;
            import java.util.StringTokenizer;
            import java.io.IOException;            
            
            public class Main {
            	public static void main(String[] args) {
            		System.out.println("Enter item qty price");
            		
            		// 2. set up readers & tokeniser
            		InputStreamReader isr = new InputStreamReader( System.in );
            		BufferedReader br = new BufferedReader(isr);
            		StringTokenizer st = null;  
            		
            		// 3. read input line
            		// note that must catch exception
            		try {
                    st = new StringTokenizer(br.readLine());
                } catch (IOException e) {
                    e.printStackTrace();
                }
            		
            		// 4. tokenise (chop up) the input into respective variables
            		// note parse methods for int, float            		
            		String item = st.nextToken();
                int qty = Integer.parseInt( st.nextToken() );
            		float price = Float.parseFloat( st.nextToken() );
            		
            		// use input variables as needed
            		System.out.println("input item is: " + item);
            		System.out.println("input qty is: " + qty);
            		System.out.println("input price is: " + price);
            	}
            }

Reusable snippets of code

- DRY principle

- divide & conquer

- better code organisation, reusability, and modularity

So far we've been putting all our code in one method called main().
And again I stress that you must always have a main() method.
Does not mean, however, that you cannot have other methods.

You certainly can, and in fact you are encouraged to,
split your logic up into other methods wherever & whenever
applicable.

WHY? To organise code, make it more reusable,
and make programs more modular and structured

WHEN? scenarios/use cases for this
- repeated code
- break problem into steps to reduce complexity

// defining function example 1
public static void sayHello() {
    System.out.println("function says hello!!");
}

// defining function example 2
public static void sayAnything(String msg) {
    System.out.println(msg);
}

// defining function example 3
public static int calcSum(int[] nums) {
    int sum = 0;
    for(int n : nums) {
        sum += n;
    }
    return sum;
}

What's an object? and what's it got to do with a class?

Class is a template a pattern a model a blueprint a mold

Cookie cutter vs cookie

An object is an instance of a class

and use them in your main program. 


                // create an instance
